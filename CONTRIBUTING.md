# Contributing to BORG Collective

Thanks for considering contributing to **BORG (Bridge for Orchestrated Reasoning Groups)**.

## Ways to Contribute

- **Feedback & Ideas** – Open an Issue with suggestions or critiques.
- **Docs** – Improve README, architecture docs, examples.
- **Code** – Help build:
  - Phase 1 prototype UI
  - Browser extension (Phase 2A)
  - Local middleware server (Phase 2B)
- **Testing** – Try early versions and report bugs.

## Getting Started

1. Check the `ROADMAP.md` to see current focus.
2. Look at open Issues labeled `good first issue` or `help wanted`.
3. Comment on an issue to say you’re working on it.

## Pull Request Guidelines

- Keep PRs focused on one logical change.
- Add or update documentation when needed.
- Include a brief description of:
  - What changed
  - Why it changed
  - Any breaking behavior

## Code of Conduct

- Be respectful and collaborative.
- Assume good faith.
- No harassment, hate, or abuse.

BORG aims to be a **global, democratic tool** for multi-AI collaboration.  
Your contributions help bring that future closer.
